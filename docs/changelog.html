2026-05-27 최신
fix tmux 같은 세션 다른 창에서 배너가 표시되지 않는 문제

is_terminal_focused()가 세션 단위로만 포커스를 판단해, 같은 tmux 세션 안에서 Claude Code가 실행 중인 창이 아닌 다른 창을 보고 있어도 "터미널 포커스됨"으로 판정되어 배너가 생략되던 문제.
클라이언트가 현재 바라보는 창·pane 번호( tmux display-message -c <client> -p '#I/#P')를 훅 실행 창·pane과 비교하도록 수정. 다른 창을 보는 중이면 배너가 정상 표시됨.

fix 터미널로 전환 시 권한 배너가 자동 허용되는 버그

권한 배너가 표시된 상태에서 사용자가 터미널로 전환하면 clearAll()이 대기 중인 permission 배너를 .dismiss로 해제하고, 훅 스크립트는 이를 "명시적 결정 없음"으로 보고 exit 0(fail-open = 자동 허용)으로 처리하던 문제.
터미널 포커스 이벤트에서 clearInfoBanners()만 호출하도록 변경. 작업완료·질문 배너는 여전히 자동 소멸되지만, 권한 배너는 사용자가 Yes/No를 명시적으로 선택할 때까지 유지됨.

fix 버그 수정 및 코드 정리 4건

① Logger 성능 — log()가 호출될 때마다 ISO8601DateFormatter()를 새로 생성하던 문제. static 프로퍼티로 캐시해 재사용.
② dead code 제거 — PanelManager.Entry.height 필드가 저장되지만 layoutPanels()에서 한 번도 읽히지 않아 제거.
③ tmux stdout 누출 — TmuxFocus.run()의 standardOutput이 리다이렉트되지 않아 tmux 명령 출력이 앱 stdout으로 누출되던 문제. Pipe()로 억제.
④ e2e 테스트 TTY walk — 테스트 스크립트의 TTY 역추적 루프에서 ppid=0 가드가 누락되어 있던 점 수정 (훅 스크립트와 일관성 확보).

fix 보안 취약점 및 버그 3건 수정

① AppleScript injection 방어 — HTTP 페이로드의 tty 필드가 검증 없이 AppleScript 문자열에 삽입돼 임의 명령 실행이 가능하던 문제. focusByTTY() 진입 전 /dev/[a-zA-Z0-9/]+ 정규식으로 TTY 경로를 검증하도록 수정.
② allowSession 이중 캐시 기록 제거 — "Session" 클릭 시 Swift 앱과 bash 훅 스크립트가 동시에 세션 캐시를 기록하던 문제. Swift 측 writeSessionCache 코드를 제거하고 bash 훅이 단독으로 처리하도록 정리.
③ 배너 상한 초과 시 NotificationStore 메모리 누수 — 배너가 20개 한도로 드롭될 때 resolve만 호출하고 remove를 누락해 pending·events 항목이 앱 재시작 전까지 남던 문제. resolve + remove 쌍으로 처리하도록 수정.
